Unhosted

Freedom from web 2.0's monopoly platforms

Unhosted is a project aimed to decentralize the web by separating apps from user data , thus, strengthening free online software development and increasing users' privacy.

To find out more, check out our proof-of-concept (not real usable code yet, please contribute if you see errors in it), read our full manifesto, and our wiki, or ask a question on our mailing list.

Benefits of being unhosted Five reasons to migrate now

The unhosted web architecture moves user data away from app servers. This has many advantages over the web2.0 architecture, both for users and for developers.

  • No server-side spying

    By letting unhosted web apps apply payload encryption to your data, you prevent your unhosted account provider from infringing on your privacy.

  • No dependency on commercial software

    As data moves away from the application, resource costs move away from the software author. This makes it feasible for non-commercial web apps to compete with commercial ones.

  • Decentralization

    More robust/resilient architecture, with no Single Point of Failure - especially if combined with modern in-browser storage techniques.

  • Data freedom

    With your data living outside the app, you can switch back and forth between apps without even exporting and importing. Better than data liberation, this is data freedom brought to the web.

  • Scalability

    Developers only pay for seeding the statics until ISPs and browsers cache them, so a very basic http server can scale up to serve a very popular web app for many users.
