Unhosted

Freedom from web 2.0's monopoly platforms

Unhosted is an Open Source project aiming to provide users with their own share of online resources, thus making it perfect for altruistic websites and for bringing public domain applications to the web.

Download the code at gitHub Read the manifesto

How does unhosted work?

A website is a specific web app, hosted on a specific server farm. There is a limited number of big centralized websites, that we all connect to. Our web has been taken hostage and monopolized.

Unhosted web apps are not hosted on one specific server farm. They are freed. The once-important server farms now become commodity infrastructure, and authors of unhosted web apps liberate the user.

Hacky Holydays!

We are pleased to announce that Unhosted's 'Hacky Holidays' competition is on! The deadline is New Year's Day 2011. If you are able to write an unhosted web app during those boring Christmas days, then you will be one of the first people in the world to have done that, and additionally, if your app is really cool, then you'll win one of three plug servers (http://www.tonidoplug.com/ or any similar one you prefer). If you don't know what unhosted is, read our manifesto first.

Get our tarball from github and play around with the 'hello world' examples. There is one Pub/Sub-model example and one messaging-model example. From there, take your inspiration and put a cool app together. If you get stuck with how to do something and want some advice, ask your questions on the mailing list and we'll answer them there. The third example is about cryptography and especially for the ubergeeks among us, but you won't need that one to win.

To make an unhosted web app work, you need to program all functionality client-side, and in a distributed architecture, so a bit of JavaScript skills come in handy, as does experience with scalability design. It's the perfect puzzle challenge for frontend developers, backend developers, sysadmins, and everyone inbetween! :) Linux-based plug servers are just about as cool as a geeky gadget can get. So we really hope you think of a cool app you could write into a distributed client-side form, and have a Hacky Holiday!

Separate categories for webdevs and geeks are on the right. You are encouraged to compete in more than one category, to increase your chances of winning one of the three desirable plug servers. Deadline: New Year's Day 2011.

WebDevs Show your cool apps!

Be the first person to write an unhosted WebApp! twitter, chat, farmville, poker, forum, webmail, whatever you can hack together with your web development and JavaScript skills.

You can use the hello-world examples from the tarball as a starting point. They include a Pub/Sub-model wappblog and a messaging-based wappmail example.

Geeks help with protocol improvements!

The UJ/0.1 protocol and the StarSign0.1 pki are very basic, and there are many things that can (and probably should) be added to them. Like support for federation, compatibility with CCN, improvement of security, designing a browser to browser discovery & friending mechanism, etcetera.
